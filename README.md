# A-Math KOTH Manager

เว็บแอปสำหรับจัดการแข่งขัน **A-Math แบบ King of the Hill (KOTH)** ที่เริ่มต้นโดยไม่มีทีม ไม่มีคู่แข่งขัน และไม่มีผลการแข่งขัน เพื่อใช้สร้างรายการใหม่ได้ทุกปี ทุกระดับชั้น และทุกจำนวนรอบที่ต้องการ

ระบบนี้พร้อมใช้งานบน **Cloudflare Workers + Cloudflare D1** โดยไม่ต้องมีเซิร์ฟเวอร์ของโรงเรียนเปิดตลอดเวลา

## ความสามารถของระบบ

- สร้างและเก็บรายการแข่งขันได้ไม่จำกัด เช่น ม.ต้น ปี 2569, ม.ปลาย ปี 2570 หรือรายการซ้อม
- เริ่มข้อมูลว่างสำหรับรายการใหม่ทุกครั้ง โดยรายการเดิมยังอยู่ครบ
- เพิ่มทีมทีละทีม หรือวางข้อมูลหลายบรรทัดจาก Excel เพื่อสร้างทีมพร้อมกัน
- เก็บรหัสทีม, Seed, โรงเรียน, ผู้เข้าแข่งขัน 2 คน, ครูผู้ควบคุม, ช่องทางติดต่อ และหมายเหตุ
- จับคู่เกมแรกได้ตาม Seed หรือสุ่มลำดับ
- จับคู่เกม KOTH ถัดไปตามอันดับปัจจุบัน และพยายามหลีกเลี่ยงคู่ที่เคยพบกันแล้ว
- รองรับจำนวนทีมคี่ด้วย BYE โดยพยายามกระจาย BYE ให้เป็นธรรม
- แก้ไขคู่ก่อนยืนยันผลการแข่งขันได้
- บันทึกผลรายโต๊ะและคำนวณ ชนะ / เสมอ / แพ้ / คะแนน / แต้มได้ / ผลต่างสะสมอัตโนมัติ
- ตั้งคะแนนชนะ, เสมอ, แพ้, BYE ได้เอง
- ตั้งเพดานผลต่างคะแนนได้ทั้งรายเกมและค่าเริ่มต้น เช่น `250,250,250,250,200`
- จัดอันดับตาม คะแนน → ผลต่างหลังคุมเพดาน → แต้มได้ → ชนะ → ชื่อทีม (เปลี่ยนลำดับรองรับได้ 2 รูปแบบ)
- สร้างรอบชิง Top 4 ได้: รอบรองชนะเลิศ, รอบชิงชนะเลิศ และชิงอันดับ 3
- รองรับกรณีคะแนนเสมอในรอบชิง โดยให้ผู้ดูแลเลือกผู้ชนะ
- เปิดหน้าตารางคะแนนสาธารณะสำหรับผู้ชมโดยไม่เปิดสิทธิ์แก้ไข
- ส่งออกตารางคะแนน CSV, ดาวน์โหลดข้อมูลสำรอง JSON และนำเข้าไฟล์สำรองเป็นรายการใหม่
- รองรับการพิมพ์ตารางคะแนนและใบประกบคู่จากเบราว์เซอร์
- มีรหัสผ่านผู้ดูแล และ session แบบ HttpOnly Cookie

## โครงสร้างเทคโนโลยี

- **Frontend:** HTML, CSS, JavaScript ล้วน ไม่ต้อง build frontend และไม่มี dependency ฝั่งหน้าเว็บ
- **Backend:** Cloudflare Worker API
- **Database:** Cloudflare D1 (SQLite แบบ serverless)
- **Authentication:** รหัสผ่านผู้ดูแล 1 ชุด เก็บเป็น Cloudflare Secret
- **Static Assets:** ถูกเสิร์ฟจาก Workers Static Assets ในโปรเจกต์เดียวกับ API

## ติดตั้งและ Deploy บน Cloudflare

### 1) เตรียมเครื่องมือ

ติดตั้ง Node.js รุ่น LTS และมีบัญชี Cloudflare แล้วเปิด Terminal ในโฟลเดอร์นี้

```bash
npm install
npx wrangler login
```

### 2) สร้างฐานข้อมูล D1

```bash
npx wrangler d1 create amath-koth-db
```

คำสั่งจะแสดง `database_id` หน้าตาประมาณ UUID ให้คัดลอกไปแทนค่าใน `wrangler.jsonc` ตรงนี้:

```jsonc
"database_id": "00000000-0000-0000-0000-000000000000"
```

> สามารถตั้งชื่อฐานข้อมูลอื่นได้ แต่ต้องใช้ชื่อเดียวกันทุกคำสั่งด้านล่าง

### 3) สร้างตารางฐานข้อมูล

```bash
npx wrangler d1 migrations apply amath-koth-db --remote
```

### 4) ตั้งรหัสลับผู้ดูแล

ตั้งรหัสผ่านที่ใช้เข้าสู่ระบบ:

```bash
npx wrangler secret put ADMIN_PASSWORD
```

ตั้งค่า secret สำหรับเข้ารหัส session (ใช้ข้อความสุ่มยาวอย่างน้อย 32 ตัวอักษร):

```bash
npx wrangler secret put AUTH_SECRET
```

ตัวอย่างสร้างข้อความสุ่มใน macOS / Linux:

```bash
openssl rand -hex 32
```

### 5) Deploy

```bash
npx wrangler deploy
```

Wrangler จะแจ้ง URL ของเว็บแอป เช่น `https://amath-koth-manager.<account>.workers.dev` เปิด URL นั้นแล้วเข้าสู่ระบบด้วย `ADMIN_PASSWORD` ที่กำหนดไว้

## ทดลองในเครื่องก่อน Deploy

1. สร้างไฟล์ `.dev.vars` จาก `.dev.vars.example`
2. กำหนดค่า `ADMIN_PASSWORD` และ `AUTH_SECRET` ของเครื่องทดสอบ
3. ใช้คำสั่ง:

```bash
npm install
npm run db:migrate:local
npm run dev
```

จากนั้นเปิด `http://localhost:8787`

## ตรวจสอบโค้ดและตรรกะหลัก

```bash
npm run check
npm test
```

ชุดทดสอบตรวจสอบการคำนวณผลต่างแบบมีเพดาน, การจับคู่ที่หลีกเลี่ยงคู่เดิม และการจัดการ BYE สำหรับจำนวนทีมคี่

## เริ่มใช้งานหลังติดตั้ง

1. สร้าง **รายการแข่งขันใหม่** และกำหนดชื่อ/ปี/สถานที่/จำนวนเกมที่วางแผน
2. ไปที่ **ทีมแข่งขัน** เพื่อเพิ่มหรือวางรายชื่อจาก Excel
3. ตรวจสอบกติกาที่ **ตั้งค่ารายการ** โดยเฉพาะคะแนนและเพดานผลต่างของแต่ละเกม
4. ไปที่ **จับคู่และบันทึกผล** เพื่อสร้างเกม KOTH เกมแรก
5. บันทึกผลให้ครบทุกโต๊ะ แล้วจึงสร้างเกมถัดไป
6. เมื่อครบ KOTH แล้ว จะเลือกสร้าง **รอบชิง Top 4** หรือจบการแข่งขันจากตาราง KOTH ได้
7. เปิด **รายงานและเผยแพร่** เพื่อเปิดหน้าตารางคะแนนสาธารณะ ดาวน์โหลดผล หรือสำรองข้อมูล

## ข้อควรรู้เรื่องกติกา KOTH ในระบบ

- เกมแรก: เลือกจับคู่ตาม Seed หรือสุ่มได้
- เกมถัดไป: เรียงอันดับปัจจุบันแล้วประกบกับอันดับใกล้กัน
- ระบบตรวจประวัติคู่แข่งขัน และจะสลับคู่ที่อยู่ถัดลงไปเพื่อเลี่ยงการพบซ้ำเมื่อทำได้
- เมื่อหลีกเลี่ยงไม่ได้ ระบบจะแสดงคำเตือนในรอบนั้น เพื่อให้ผู้ดูแลตัดสินใจแก้ไขคู่ด้วยตนเองก่อนเริ่มแข่ง
- BYE เริ่มเลือกจากทีมอันดับล่างที่มีจำนวน BYE น้อยที่สุด เพื่อไม่ให้ทีมเดิมได้รับ BYE ซ้ำโดยไม่จำเป็น
- แมตช์ BYE ได้คะแนนตามค่า `คะแนน BYE` แต่ไม่เพิ่มแต้มได้/แต้มเสีย/ผลต่างคะแนน
- ผลต่างคะแนนของแต่ละแมตช์ถูกจำกัดด้วยค่าเพดานของเกมนั้นก่อนรวมอันดับ
- รอบชิงไม่ถูกรวมกับคะแนน KOTH เพื่อรักษาอันดับ KOTH เดิมอย่างชัดเจน

## เปลี่ยนโดเมนเอง

หลัง Deploy แล้ว สามารถผูก Custom Domain ได้ที่ Cloudflare Dashboard ของ Worker นั้น โดยไม่ต้องแก้โค้ด ระบบใช้ URL เดียวกันทั้งหน้าเว็บและ API จึงไม่มีปัญหา CORS ระหว่างหน้าเว็บกับฐานข้อมูล

## Deploy ที่ผู้ให้บริการอื่น

โค้ดนี้ทำให้ **Cloudflare** เป็นเป้าหมายหลัก เพราะใช้งาน D1 และ Workers binding โดยตรง หากต้องการย้ายไป Vercel, Netlify หรือเซิร์ฟเวอร์อื่น สามารถนำ `public/` ไปใช้เป็นหน้าเว็บได้ แต่ backend ใน `src/worker.js` ต้องเปลี่ยนส่วน D1 binding (`env.DB`) เป็นฐานข้อมูลของผู้ให้บริการนั้น เช่น PostgreSQL / Supabase / Neon ก่อนใช้งานจริง
